# Proof intake — reviews & case studies

Fill in the real facts below and hand it back. I'll turn each block into
polished testimonial copy and a case study, then load them into the Proof
CMS (`/admin` → Proof) so they appear on the home page and `/work`.

Rules I'm holding to, so nothing backfires:

- **Only real clients, real outcomes.** A number goes on the site only if
  it happened. "Roughly" is fine — "3–4× more orders" beats a fake "412%".
- **A quote needs a real person** who actually said it (or something close
  that they'll approve). Name + role, e.g. _"Aarav Shah, Founder"_.
- Leave a field **blank** if you don't have it — I'll write around it
  rather than invent it. A case study with 2 real metrics still converts;
  a fake third one gets the page penalised.

You don't need to write nicely — bullet points and rough numbers are
enough. I do the wording.

---

## How to fill each block

- **What they came to us for** — the problem, in your words.
- **What we did** — the services/work, briefly.
- **Result** — the real outcomes. Numbers if you have them, plain if not.
- **Timeframe** — e.g. "4 months".
- **Quote** — a real sentence or two from the client (optional but strong).
- **Quote by** — real name + role, e.g. "Priya Nair, Marketing Lead".
