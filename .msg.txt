Rebuild admin dashboard as a business overview

The old dashboard led with team members, active sessions and total audit
events - true facts that answer none of an owner's questions, and no
demand data at all.

The new dashboard is demand-first with a 7/30/90-day range switch, every
headline compared against the immediately preceding window of equal
length:

- KPI tiles: enquiries, pageviews, enquiry rate and newsletter signups,
  each with a sparkline and a period-over-period delta.
- Two separate daily columns for enquiries and pageviews - not one
  dual-axis chart, since the scales differ.
- Breakdown bars: enquiry source, services asked for, how they found us,
  most-viewed pages, top referrers, and the status pipeline.
- Latest enquiries list linking to /admin/leads.
- Site operations kept but demoted below the fold.

lib/dashboard.ts aggregates: daily buckets via one date_trunc query per
series, zero-filled so a gap reads as a real zero. Deltas return null
against a zero baseline and the UI prints "no baseline" rather than a
fabricated percentage. Rate is enquiries over pageviews, labelled as such
because we count views not sessions.

Charts follow the dataviz method: single-hue sequential throughout,
identity carried by the row label not colour, deltas stated with an icon
and sign not colour alone.

Verified with seeded data across all four sources then with it removed:
range switch repaints, deltas compute, the zero-data state shows a
0.00 rate with no divide-by-zero and no NaN, empty panels show their
messages, dark mode and mobile both clean. Seed data deleted after.
